# Requirements Spec — 패독 코리아 (Paddock Korea)

## 1. Actors

| Actor | Description |
|---|---|
| Primary User | 닉네임을 설정하고 채팅·밈 기능으로 소통하는 한국의 F1 팬 |
| Admin | 이번 MVP에서는 제외. F1 101·경기 일정·순위 데이터는 정적 데이터로 처리한다. |

---

## 2. Main Use Cases

### UC-001. 팀 선택 및 닉네임 설정 (온보딩)

- **Actor:** Primary User
- **Goal:** 선호 팀과 닉네임을 설정하여 팀 채팅방과 전체 채팅방에 참여할 수 있는 상태가 된다.
- **Precondition:** 사용자가 처음 앱에 진입했거나 설정 정보가 없는 상태이다.
- **Main Flow:**
  1. 사용자가 앱 첫 화면에서 닉네임을 입력한다.
  2. 10개 팀(Ferrari, Red Bull, Mercedes 등) 중 하나를 선택한다.
  3. 설정 정보가 로컬에 저장되고, 메인 화면(전체 채팅)으로 이동한다.
- **Alternative Flow:** 이미 설정 정보가 저장된 사용자는 온보딩을 건너뛰고 메인 화면으로 진입한다.
- **Result:** 닉네임·선택 팀이 저장되며, 이후 채팅 메시지에 팀 색상과 닉네임이 표시된다.

---

### UC-002. 전체 채팅 참여 (The Main Straight)

- **Actor:** Primary User
- **Goal:** 팀에 상관없이 모든 팬과 실시간으로 소통한다.
- **Precondition:** 온보딩이 완료된 상태이다.
- **Main Flow:**
  1. 사용자가 하단 내비게이션에서 The Main Straight 탭을 선택한다.
  2. 최근 채팅 메시지 목록이 시간순으로 표시된다.
  3. 사용자가 입력창에 메시지를 작성하고 전송 버튼을 누른다.
  4. 메시지가 채팅창에 즉시 표시되며, 다른 참여자의 메시지도 자동으로 갱신된다.
- **Alternative Flow:** 메시지 입력창이 비어 있으면 전송 버튼이 비활성화된다.
- **Result:** 메시지가 전체 채팅방에 등록되어 모든 사용자에게 표시된다.

---

### UC-003. 팀별 채팅 참여 (The Garage)

- **Actor:** Primary User
- **Goal:** 자신이 선택한 팀 팬들과 전용 채팅방에서 소통한다.
- **Precondition:** 온보딩이 완료된 상태이다.
- **Main Flow:**
  1. 사용자가 The Garage 탭을 선택한다.
  2. 자신의 선택 팀 전용 채팅방이 열리며 최근 메시지 목록이 표시된다.
  3. 사용자가 메시지를 작성하고 전송 버튼을 누른다.
  4. 메시지가 해당 팀 채팅방에 즉시 표시되며 같은 팀 팬의 메시지만 나타난다.
- **Alternative Flow:** 메시지 입력창이 비어 있으면 전송 버튼이 비활성화된다.
- **Result:** 메시지가 팀 전용 채팅방에 등록된다.

---

### UC-004. 밈 업로드 (Meme Box)

- **Actor:** Primary User
- **Goal:** 채팅과 별도의 밈 공유 공간에 이미지 URL과 캡션으로 밈을 게시한다.
- **Precondition:** 온보딩이 완료된 상태이다.
- **Main Flow:**
  1. 사용자가 Meme Box 탭에서 밈 올리기 버튼을 누른다.
  2. 이미지 URL과 캡션(선택)을 입력한다.
  3. 게시 버튼을 누른다.
  4. 밈이 Meme Box 피드 최상단에 표시된다.
- **Alternative Flow:** 이미지 URL이 비어 있거나 유효하지 않으면 게시 버튼이 비활성화되거나 오류 안내가 표시된다.
- **Result:** 밈이 Meme Box 피드에 등록된다.

---

### UC-005. 밈 조회 및 반응

- **Actor:** Primary User
- **Goal:** Meme Box에서 다른 팬이 올린 밈을 보고 좋아요로 반응한다.
- **Precondition:** Meme Box에 밈이 하나 이상 존재한다.
- **Main Flow:**
  1. 사용자가 Meme Box 탭을 열면 밈 피드가 최신순으로 표시된다.
  2. 사용자가 원하는 밈의 좋아요 버튼을 누른다.
  3. 해당 밈의 좋아요 수가 즉시 반영된다.
- **Alternative Flow:** 밈이 없으면 "아직 올라온 밈이 없습니다" 안내가 표시된다.
- **Result:** 좋아요 반응이 저장되고 수치가 업데이트된다.

---

### UC-006. F1 입문 가이드 조회 (F1 101)

- **Actor:** Primary User
- **Goal:** F1 용어 사전과 경기 진행 방식을 확인하여 F1을 처음 접하는 팬도 이해할 수 있다.
- **Precondition:** 없음.
- **Main Flow:**
  1. 사용자가 F1 101 탭을 선택한다.
  2. 카테고리 목록(용어 사전 / 경기 방식 / 레이스 위켄드 구조)이 표시된다.
  3. 항목을 선택하면 카드 형식의 상세 설명이 표시된다.
- **Alternative Flow:** 없음.
- **Result:** 사용자가 F1 기초 지식을 습득한다.

---

### UC-007. 경기 일정 확인

- **Actor:** Primary User
- **Goal:** 다가오는 레이스 위켄드 일정을 KST 기준으로 확인한다.
- **Precondition:** 없음.
- **Main Flow:**
  1. 사용자가 Pit Wall 탭의 경기 일정 섹션에 진입한다.
  2. 현재 시즌의 레이스 위켄드 목록이 KST 기준 날짜·시간과 함께 표시된다.
  3. 가장 가까운 다음 레이스가 상단에 강조 표시된다.
- **Alternative Flow:** 없음.
- **Result:** 사용자가 다음 경기 일정을 확인한다.

---

### UC-008. 챔피언십 순위 확인

- **Actor:** Primary User
- **Goal:** 현재 시즌 드라이버·컨스트럭터 챔피언십 순위를 확인한다.
- **Precondition:** 없음.
- **Main Flow:**
  1. 사용자가 Pit Wall 탭에서 드라이버 순위 또는 컨스트럭터 순위 탭을 선택한다.
  2. 순위, 이름, 포인트가 정렬된 목록으로 표시된다.
- **Alternative Flow:** 없음.
- **Result:** 사용자가 현재 시즌 순위를 파악한다.

## 3. Functional Requirements

| ID | Requirement | Priority |
|---|---|---|
| FR-001 | 사용자는 닉네임과 선호 팀을 설정하여 서비스에 참여할 수 있다. | Must |
| FR-002 | 사용자는 전체 채팅방(The Main Straight)에서 메시지를 보내고 받을 수 있다. | Must |
| FR-003 | 사용자는 팀 전용 채팅방(The Garage)에서 같은 팀 팬들과 메시지를 주고받을 수 있다. | Must |
| FR-004 | 채팅 메시지에는 닉네임과 팀 색상이 함께 표시된다. | Must |
| FR-005 | 채팅 메시지 목록은 새 메시지가 생기면 자동으로 갱신된다. | Must |
| FR-006 | 사용자는 Meme Box에 이미지 URL과 캡션으로 밈을 게시할 수 있다. | Must |
| FR-007 | 사용자는 Meme Box에서 밈 피드를 최신순으로 조회할 수 있다. | Must |
| FR-008 | 사용자는 밈에 좋아요 반응을 할 수 있으며, 좋아요 수가 표시된다. | Must |
| FR-009 | 사용자는 F1 입문 가이드(용어 사전, 경기 방식)를 카드 형식으로 조회할 수 있다. | Must |
| FR-010 | 사용자는 현재 시즌 레이스 위켄드 일정을 KST 기준으로 확인할 수 있다. | Must |
| FR-011 | 사용자는 드라이버 및 컨스트럭터 챔피언십 순위를 확인할 수 있다. | Must |
| FR-012 | 사용자는 Meme Box에서 좋아요 수 기준 인기순으로 밈을 정렬할 수 있다. | Should |
| FR-013 | 사용자는 설정에서 닉네임 또는 선택 팀을 변경할 수 있다. | Nice |

## 4. Non-functional Requirements

| ID | Requirement |
|---|---|
| NFR-001 | 모바일 화면(375px 이상)에서도 핵심 기능을 사용할 수 있어야 한다. |
| NFR-002 | 버튼과 입력 필드는 접근 가능한 이름(aria-label 또는 label)을 가져야 한다. |
| NFR-003 | 민감한 정보(API Key 등)는 GitHub 저장소에 커밋하지 않는다. |
| NFR-004 | 로그인 없이 닉네임·팀 정보를 로컬 스토리지에 저장하여 단일 사용자 기준으로 구현한다. |
| NFR-005 | 페이지 초기 로딩은 3초 이내여야 한다. |
| NFR-006 | 서비스 UI는 한국어를 기본 언어로 사용한다. |

## 5. Acceptance Criteria

### AC-001. 팀 선택 및 닉네임 설정

Given 사용자가 처음 앱에 진입하여 설정 정보가 없는 상태일 때  
When 닉네임을 입력하고 팀을 선택하면  
Then 설정이 저장되고 메인 채팅 화면으로 이동한다.

---

### AC-002. 전체 채팅 메시지 전송

Given 온보딩이 완료된 사용자가 The Main Straight 탭에 있을 때  
When 메시지를 입력하고 전송 버튼을 누르면  
Then 해당 메시지가 닉네임·팀 색상과 함께 채팅창에 즉시 표시된다.

---

### AC-003. 팀별 채팅 메시지 전송

Given 온보딩이 완료된 사용자가 The Garage 탭에 있을 때  
When 메시지를 입력하고 전송 버튼을 누르면  
Then 해당 메시지가 팀 전용 채팅방에만 표시된다.

---

### AC-004. 밈 업로드

Given 사용자가 유효한 이미지 URL을 입력했을 때  
When 게시 버튼을 누르면  
Then 해당 밈이 Meme Box 피드 최상단에 표시된다.

---

### AC-005. 밈 좋아요

Given Meme Box에 밈이 하나 이상 존재할 때  
When 사용자가 좋아요 버튼을 누르면  
Then 해당 밈의 좋아요 수가 1 증가하여 즉시 표시된다.

---

### AC-006. F1 101 가이드 조회

Given 사용자가 F1 101 탭에 진입했을 때  
When 카테고리 항목(용어 사전 / 경기 방식 등)을 선택하면  
Then 해당 항목의 상세 설명이 카드 형식으로 표시된다.

---

### AC-007. 경기 일정 확인

Given 사용자가 경기 일정 화면에 진입했을 때  
When 화면이 로드되면  
Then 다가오는 레이스 위켄드 일정이 KST 기준 날짜·시간과 함께 표시되며, 가장 가까운 일정이 상단에 강조된다.

---

### AC-008. 챔피언십 순위 확인

Given 사용자가 Pit Wall 탭에 진입했을 때  
When 드라이버 또는 컨스트럭터 탭을 선택하면  
Then 현재 시즌 순위, 이름, 포인트가 정렬된 목록으로 표시된다.

---
